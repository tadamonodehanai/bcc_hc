初期9本で読まれ方の感触をつかんだあと、次に向かうべきは関係者の継続協力・カンパニー横断・連載化が必要な、重量のある9本。「ＢＣＣのヘルスケア」が、カンパニーとして発信できている状態に到達するためのフェーズ。

Axis 01 / Open up

ひらく

単発の開示から、継続ドキュメンタリーへ。

初期では「目線」を開いた。中期では「時間軸」を開く。導入から3ヶ月の密着、判断ログ、部門横断座談 ── どれも継続協力が前提のため、初期で関係性を作ったあとに動かす。

No.04
FORMAT / DOCUMENTARY
時間の中で、何が起こったかをひらく。

介護DX伴走、3ヶ月密着。

特定の1施設の導入プロセスを継続取材し、月1更新で連載化。事例ページとは別レイヤーの、ドキュメンタリー型の物語として読ませる。

事例集の「Before / After」では落ちる、途中で起こったことを開く。施設との継続的な合意形成が前提なので中期に置く。読者にとっては「自分の施設でも起こりそうな話」として強く接続する記事になる。

Primary Reader
介護施設運営者副：自治体担当者
Format
月1更新 / 3〜4本連載
Resource
施設の継続協力（重）
No.05
FORMAT / RETROSPECTIVE
支点を見つける、その判断を振り返る。

判断ログ、分岐点の選び方。

過去の伴走支援で「なぜA案ではなくB案を選んだか」を、当時の担当者と振り返るシリーズ。事例の表面では見えない、意思決定の質を開示する。

事例記事が「結果」を語るなら、これは「結果に至るまでの判断」を語る。Mission「支点を見つける」の最も直接的な体現記事。導入を検討する施設・自治体にとって、最も信頼形成に効く形式になる。

Primary Reader
介護施設運営者副：自治体担当者
Format
過去事例の振り返り / 連載
Resource
担当者＋当時の顧客（中重）
No.06
FORMAT / ROUNDTABLE
4つの現場の違いを、ひとつのテーブルで開く。

部門横断スタッフ座談会。

介護施設担当・一般企業担当・自治体担当・個人向け担当が一堂に会する座談会。「現場が違うと、何が違うか」を語り合うことで、4つの顧客像をまたぐカンパニー全体の理念が言葉になる。

IT営業カンパニーの「人物紹介」型インタビューとの、最大の差別化ポイントになる企画。一人称ではなく、複数の現場担当者の対話を通じて、カンパニーとしての一貫性を可視化する。中期フェーズの中で、最もアイデンティティ形成に効く1本。

Axis 02 / Connect

つなぐ

立場の橋を、より多くの層に渡す。

初期では「企業 × 施設」「現場の感情」をつないだ。中期は自治体・住民・カンパニー横断まで対象を広げる。Concept「健やかさは、めぐる」のスケールが、ここで見えてくる。

No.04
FORMAT / FIELD REPORT
企業と地域と住民、三者の交点をつなぐ。

リビングラボの、取材連載。

自治体 × 企業 × 住民が交わる場を、継続取材。リビングラボの取り組みを「行政施策」ではなく「生活の中で何が起きたか」として記録する。

1記事あたりの情報密度が高く、自治体読者の信頼を獲得する中核コンテンツ群になる。同時に、企業読者にとっては「自社商品が地域でどう使われるか」のイメージが具体化する記事として機能する。

Primary Reader
自治体担当者副：一般企業
Format
現地レポート / 連載

自治体＋企業協力（重）
No.05
FORMAT / DIALOGUE
分断が起きる「場所」を、三者で名指す。

三者対談、健やかさはどこで止まるか。

自治体・施設・企業の3者で、Conceptに書かれた「健やかさが止まる場所」を議論する。理念を抽象論にせず、現実の課題地図として可視化する企画。

Concept「健やかさは、めぐる」を、逆向きから検証する企画。「めぐらせる」前に「どこで止まっているか」を3者の視点で名指すことで、理念が現実に着地する。中期の中で、最もブランド理念を強化する1本。

三者対談 / 大型1本

No.06
FORMAT / DIALOGUE
社内に分かれていたものを、つなぐ。

カンパニー横断対談、IT営業 × ヘルスケア。

IT営業アウトソーシングカンパニーの担当者と、ヘルスケアの担当者が、「DX推進」という共通テーマで対談。社内で見えていなかった接点を可視化する。

社外向け発信と、社内向けインターナルブランディングを兼ねる企画。ＢＣＣグループ全体としての物語をつくる起点。両カンパニーの読者層が違うため、片側だけでは届かない層にもリーチできる。

Axis 03 / Deliver

届ける

運用が回るまでの、設計と学びを届ける。

初期では「導入のための地図」を渡した。中期は導入後に焦点を移す。失敗と修正、学びのオンライン化、そして次の3年へのビジョン公開まで含む。

No.04
FORMAT / REPORT
構想で終わらせない、その「終わり方」を見せる。

導入後6ヶ月、運用定着レポート。

実装後に、何が回り、何が回らなかったかを、失敗と修正も含めて開く。導入事例ページでは見せない部分まで踏み込む。

Mission「やり切る」に対して、最も誠実な記事になる。他社のDX事例との読後感の差を、決定的にここで作る。失敗を開示するためには社内の合意も必要で、リスクは小さくないが、それを越えた先に強いブランドがある。

No.05
FORMAT / DESIGN ESSAY
学びを、デジタルで届ける設計を語る。

学びのオンライン化、設計論。

高齢者健幸サポーターのeラーニング設計を例に、学びをデジタルで届ける設計論を語る。介護DXと教育を、ひとつの設計思想で接続する。

介護DXと「学び」が同じ軸の上で動いていることを、設計論として語り切る1本。教育コンテンツの発信を強化したい現場と、DX推進の文脈をひとつにつなぐ、足並みを揃える戦略の中核記事。

No.06
FORMAT / VISION
届ける先を、3年先まで宣言する。

介護DXの、次の一手。

導入が終わった施設の「その先」。ヘルスケアカンパニーがカンパニーとして、次の3年で何を提供するかの中期ビジョン公開。

CEOまたはカンパニー長クラスのオピニオン記事として展開。これまでの記事の積み上げの上に、「だから、私たちはこう動く」を宣言する。中期フェーズの締めくくりであり、次のフェーズへの入口になる1本。

●Closing

18本を経て、
「カンパニーとして」が立ち上がる。

初期9本は、3軸が機能するかを軽い投資で検証するフェーズだった。中期9本は、カンパニーを横断し、関係者の継続協力を引き出し、ブランドの土台を厚くするフェーズ。

特に中期の中で「Company-wide Identity」とラベルした3本（部門横断座談・カンパニー横断対談・介護DX次の一手）は、すべてカンパニー全体のアイデンティティを言語化する役割を持つ。ここまで来て初めて「ひとつのカンパニーとして」の発信ができている状態になる。

その先、Phase 03（6ヶ月以降）では、コンテンツが自走しはじめる。読者からの反応、現場からの企画提案、外部メディアからの取材依頼。編集が「打ち手」から「対応」に変わる段階。そのとき、すでに18本の中に答えのほとんどが書かれている。
