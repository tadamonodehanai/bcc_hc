Mission「支点を見つけ、重心を支え、人生を整える」と、Concept「健やかさは、めぐる」を、コンテンツの設計図に翻訳する。
3つの行動（ひらく・つなぐ・届ける）を軸に、初期フェーズ9本を整理しました。

01Why this strategy

なぜ「ひらく・つなぐ・届ける」を、
コンテンツの軸にするのか。

理念にすでに書かれている3つの行動を、そのまま発信の設計に重ねる。理念とコンテンツがズレないこと、それ自体が一貫性のあるブランドになる。

Context

ヘルスケアカンパニーは、介護DXを中核に、教育やマーケティングを並走させていく段階にある。一方、社内には人材派遣を中核にするIT営業アウトソーシングカンパニーがあり、「スタッフインタビュー」が機能している。

しかし、同じ手法をそのまま持ち込むと「人柄はわかるが、事業の中身がぼやける」というズレが起きる。売っているものの性質が違うから。

そこでヘルスケアでは、インタビューの問いを「人物紹介」から「現場の判断ロジックの開示」へ転換する。同時に、4つの顧客像（介護施設・一般企業・自治体・個人）に分散したコミュニケーションを、3軸の編集ポリシーで束ねる。

このページは、そのための初期9本を、社内で共通認識化するための地図です。

02Roadmap

初期 + 中期の、
2フェーズで積み上げる。

本数を埋めるためではなく、軸が機能するかを検証してから積み上げるための分け方。このページは初期9本のみを扱います。

Phase 01 / 0–3 months

初期フェーズ ── 手元の素材で、方向を検証する。

すでに手元にある素材（既存事例・ウェビナー登壇内容・社内に蓄積された担当者の知見）を中心に、軽い投資で発信の方向を検証する6本＋3本＝9本。失敗しても損失が小さく、読まれ方の感触をつかむ期間。

このページで扱う範囲

Phase 02 / 3–6 months

中期フェーズ ── 初期の手応えを、積み上げ型へ。

関係者の協力、継続取材、社内横断が必要な、より重量のある9本。カンパニー全体を巻き込む企画はこのフェーズに置く。「カンパニーとしての発信ができている状態」は、ここで初めて成立する。

Axis 01 / Open up

ひらく

閉じている判断と知見を、外側に開く3本。

「人物紹介」ではなく「現場の判断ロジックの開示」へ。担当者が何を見て、何を選んでいるかを開くことで、事業の解像度が上がる。IT営業カンパニーのスタッフインタビューとは別物として設計する。

No.01
FORMAT / INTERVIEW
支点を見つける、その目線そのものを開く。

現場で、最初に見るもの。

介護DXコンサル担当者が、施設に入った最初の30分で何を観察し、何を質問しているか。技術や提案の話に入る前の「目線」を開示する。

表面の課題解決ではなく、現場をどう読むかから開示することで「この会社は表面で判断していない」が伝わる。担当者1人あたり1時間の取材で、3〜5本のショート連載化が可能。最も低コストで、最も差別化に効く第一弾。

Primary Reader
介護施設運営者副：自治体担当者
Format
1時間取材／3〜5本連載
Resource
担当者の協力のみ（軽）
No.02
FORMAT / REPORT
制度の中にとどまっている課題を、外へ。

ウェビナーの、裏側で起きていたこと。

4月開催「介護DX推進に向けた課題抽出ウェビナー」で実際に抽出された課題を、施設名を伏せて開く。当日参加できなかった層に届ける。

すでに登壇用に整理された素材があるため、ライティングだけで成立する低コスト案。同時に「現場の生の課題」というコンテンツ価値が高く、検索でも入ってきやすい。初期の検証用としても、リード獲得用としても両立する。

Primary Reader
介護施設運営者副：一般企業
Format
レポート記事 / 1本
Resource
既存素材リライト（最軽）
No.03
FORMAT / INTERVIEW
人の中にあるものを、外側へひらく。

レク広場の、中の人。

4,000点以上のレク素材を、誰がどんな基準で選び・磨いているか。コンテンツ事業の編集現場を、設計思想として開示する。

この企画をあえて「届ける」ではなく「ひらく」に置くのが要点。素材の使い方ではなく、選ぶ基準の方を開くことで、レク広場が「単なる素材集」ではなく「学びの設計思想を持つ事業」だと伝わる。介護関係者の指名検索を強化する効果も期待できる。

Resource
編集チーム取材（中軽）
Axis 02 / Connect

つなぐ

分断された立場を、ひと続きの物語にする3本。

Valueに書かれた「企業と生活者」「地域と個人」「仕組みと感情」── 特に「仕組み × 感情」を編集の主軸に置く。介護DXのコンテンツが、他社の「導入効果◯％」型と決定的に違う読後感を生む鍵。

No.01
FORMAT / REWRITE
仕組みと感情、両面を一本に。

事例を、もう一度書き直す。

既存の伴走事例3本（ラヴィーナ・日晴金属・NEC）を、「仕組みの話」と「感情の話」の二面で書き直す。同じ事例でも、見えるものが変わる。

素材はすでに公開されているので、最も早く動かせる検証用記事。Valueの「仕組みと感情」を、社外向けに最初に証明する1本。同時に、社内の編集ポリシーが「ここが軸だ」と腹落ちする教材にもなる。

Primary Reader
一般企業副：介護施設運営者
Format
事例リライト / 3本
Resource
既存素材＋追加取材なし（最軽）
No.02
FORMAT / DIALOGUE
企業と生活者を、直接つなぐ。

企業 × 施設長、30分の対談。

パートナー企業の担当者と、その商品を導入した施設長の2人対談。30分取材の軽量フォーマットで、連載化を前提に設計する。

商品を売る企業側と、現場で使う施設側を、同じテーブルで語らせることで、どちらの読者にも届く構造になる。30分という軽さが運用継続性を生み、ヘルスケアの取扱パートナーが増えるたびに連載が育つ。

Primary Reader
一般企業副：介護施設運営者
Format
30分対談 / 連載
Resource
パートナー協力（中軽）
No.03
FORMAT / COLUMN
仕組み × 感情、その「感情」の方を集める。

数字には、出てこない変化。

現場スタッフ・利用者・家族の、数字には現れない感情の変化を集めた短編コラム集。連載として小さく、長く育てる。

「DXで時間が◯%増えた」ではなく、「利用者と話す時間が増えた」のような小さな変化を集めると、つなぐ軸では特別な読後感が生まれる。Mission「人生を整える」の体現記事として、長期で資産になる。

Primary Reader
介護関係者（個人）副：介護施設運営者
Format
短編コラム / 連載
Resource
現場ヒアリング（中）
Axis 03 / Deliver

届ける

介護DX × 学び を、実装まで運ぶ3本。

「導入事例」では足りない、というのが届ける軸の前提。運用が回りだすまでの設計を見せる。教育コンテンツ（レク介護士・健幸サポーター）は、ここで自然に同じ軸に乗る。

No.01
FORMAT / KNOWLEDGE
実装するための、最初の地図を渡す。

介護DX、導入前後の判断ポイント。

導入「前 / 中 / 後」の3フェーズで、施設側が押さえておくべき判断ポイントを整理した実用ナレッジ。資料請求への入口として機能させる。

営業活動に直結する1本。社内で何度も語られてきた「導入前に決めておけばよかったこと」を一度きちんとパッケージ化する。ダウンロード資料化することで、リード獲得の起点にもなる。届ける軸の中で最も実利的な記事。

Primary Reader
介護施設運営者副：自治体担当者
Format
ナレッジ記事＋DL資料
Resource
社内知見の整理（中）
No.02
FORMAT / INTERVIEW
「学び」を、届ける軸の主役にする。

レクリエーション介護士、学びの設計。

4万人を支えてきた教育プログラムの裏側を、学習者目線と事業者目線の両方で読める形に。「学び」軸が動き始める起点となる1本。

介護DXと教育コンテンツを同じ「届ける」軸で接続する狙い。別軸として扱わない。学びの設計思想を語ることで、レク介護士という資格の存在意義が、業界課題の文脈で再定義される。教育部門のブランディングにも効く。

Primary Reader
介護関係者（個人）副：介護施設運営者
Format
設計者インタビュー / 1本
Resource
教育チーム協力（中）
No.03
FORMAT / EXPLAINER
制度を、現場で機能する形まで噛み砕く。

介護報酬改定 × 現場の運用。

2024年改定の生産性向上要件が、現場運用にどう影響するか。ＢＣＣの導入支援とどう接続するかまで踏み込む実用解説。

SEO・指名検索の両面で効く集客性の高い記事。制度解説の記事は他社にもあるが、「ＢＣＣの支援と接続する」視点まで踏み込むのは独自性が高い。届ける軸の中で、最も検索流入の起点になりやすい1本。

Primary Reader

Format
解説記事 / 1本

03Rollout

9本を出す前に、
「3本」で軸が機能するかを見る。

このロードマップの肝は、9本を順番に出すことではなく、各軸の「No.01」を1本ずつ、合計3本だけ先に出すこと。3本で「ひらく・つなぐ・届ける」が読者に伝わるかを検証する。

初期の3本（各軸No.01）は、ここを意識して選んでいます。──

First / Open up
現場で、最初に見るもの

表面の話に入る前の目線を、最初に開く。「この会社は表面で判断しない」の宣言。

First / Connect
事例を、書き直す

仕組みと感情の二面で書く編集ポリシーを、最低コストで証明する。

First / Deliver
介護DX、導入前後の判断

営業に直結し、リード獲得にも効く実用ナレッジで、実装姿勢を示す。
