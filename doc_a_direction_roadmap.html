ドラフト
§ 01　現状認識

広場が抱える、
5つの構造的課題。

広場は立ち上げから15年、7万人の会員基盤と一定の業界認知を持つ一方で、事業としては5つの構造的課題を抱えている。これらは個別の問題ではなく、互いに連鎖して「次の手」を打ちにくくしている。

01
事業単体での売上がない。 運用コスト＞売上の状態が常態化。バナー広告と単発アンケートに依存しており、収益構造が不安定。
02
15年前のプラットフォーム。 サイト基盤が古く、効果的な会員管理・データ収集・分析ができていない。改修コストは事業規模に対して大きい。
03
素材の品質がバラバラ。 15年間の蓄積によって生まれた品質格差。一部の高品質コンテンツが利用の大半を支えているパレート構造の可能性。
04
介護レクの資格制度的優先順位の低下。 業界全体でのレクリエーションの位置づけが弱まっており、施設の投資意欲を引き出しにくい。
05
会員のアクティブ数が可視化できていない。 7万人会員のうち、実際に動いている人と非アクティブな人の区別がつかない。誰に何を届けているかが見えない。
§ 02　戦略的転換

事業の組み立て方を、
4つの観点で転換する。

課題に個別対応するのではなく、まず「事業の組み立て方そのもの」を転換する。これがすべての判断の前提になる。

SHIFT 01 — Architecture Web と LINE の役割を、
ストックとフローに分業する。 古いWebプラットフォームを無理にリプレイスせず、ストック資産として残す。LINEを行動データ収集と日常接点のフローとして新設し、両者を補完関係にする。
Web 一本依存 → Web × LINE 分業
SHIFT 02 — User Reality ユーザー像の前提を、
個人ベースに置き直す。 広場の利用者は法人や組織として動いているのではなく、現場の介護職個人が自己責任で使っている可能性が高い。施設契約モデルを前提に置かず、個人ユーザーの実像を起点にする。
法人想定 → 個人ユーザー前提
SHIFT 03 — Monetization 対価の出所を、
業界外に求める。 介護業界に直接の支払い余力は薄い。「介護現場の個人にリーチしたい第三者」から対価を取る構造、または施設が他経費科目で支出できる仕組みを設計する。単純な有料化は最初から候補から外す。
介護業界内 → 業界外スポンサー
SHIFT 04 — Discipline まず価値を加える前に、
スリム化を経営する。 新しい機能を足す前に、現在ある資産を「残す・削る・磨く」で仕分ける。スリム化によって生まれた余力を、磨く対象に集中投資する。「やめる勇気」を経営判断として明文化する。
足し算経営 → 引き算 → 磨き込み
§ 03　全体ロードマップ

観察期と本実行期に分け、
14期内で完結させる。

公式LINEの再開と並行して、まず3〜5ヶ月の観察期で材料を集める。BCC 14期（2026.10〜2027.09）に入ってから、仕分け・磨き・発信の3フェーズを順に実行する。全期間で4つの動詞「観る・仕分ける・磨く・魅せる」を貫く。

観察期（13期末） BCC 14期 — 本実行期
PHASE 0 観る 観察と仮説検証。
14期戦略書を最終化する。 2026.05 — 09　(5M)
PHASE 1 / Q1 仕分ける 残す・削る・磨くを意思決定。スリム化を実行する。 2026.10 — 12　(3M)
PHASE 2 / Q2-3 磨く 残した価値に集中投資。検証ベースで新収益のタネを試す。 2027.01 — 06　(6M)
PHASE 3 / Q4 魅せる 14期成果を可視化。15期投資判断の材料を整える。 2027.07 — 09　(3M)
PHASE 0 観る 2026年5月 — 9月
（5ヶ月、観察期）

14期の戦略を「思いつき」ではなく「証拠」で組み立てるための材料を集める。

主要アクション
  • 公式LINEの再開と最初の3ヶ月運用（既存12週間プランを実装）
  • 既存7万人会員のアクセスログ・利用パターンの棚卸し
  • 利用施設5〜10件、ヘルスケア企業／自治体3〜5社へのヒアリング
  • 競合（WAN-かいご、介護アンテナ）の再ベンチマーク
  • 5つの測定ドメインによる仮説検証（Document B 参照）
アウトプット
  • LINE運用3ヶ月レポート
  • ニーズ仮説検証レポート（5ドメイン分）
  • 14期戦略書（最終版）
判断ゲート（9月末） 戦略書をもって経営承認を取得。Phase 1で何を削るかの初期リストを確定し、14期スタートに接続する。
PHASE 1 仕分ける 2026年10月 — 12月
（3ヶ月、14期 Q1）

14期スタートに合わせて、残すもの・削るもの・磨くものを明確に意思決定し、スリム化を実行する。最も心理的に難しいフェーズ。

主要アクション
  • 「残す」「削る」「磨く」の最終決定と、社内合意の取得
  • 不採算オペレーション・低利用コンテンツの段階停止
  • 削った分の人時を、磨く対象へ再配分
  • 新コンセプトの社内発表 → 既存会員告知 → 業界への発信
  • WebサイトとLINEの役割分担を明文化
アウトプット
  • スリム化計画書と実行報告
  • 新コンセプト発表資料（社内向け／対外向け）
  • 整理後のサービスマップ
判断ゲート（12月末） スリム化のコスト削減効果とユーザー反応を経営層に報告。Phase 2の集中投資対象を最終確定する。
PHASE 2 磨く 2027年1月 — 6月
（6ヶ月、14期 Q2-Q3）

残したコアサービスに集中投資して、見える価値を作る。ここで初めて新しい収益のタネを小さく試す。

主要アクション
  • コアコンテンツの品質統一とブランディング再整備
  • LINE × Web の動線最適化、ロイヤルユーザー層の顕在化
  • EBR（科学的根拠付きレク）の試験運用（記事・事例・白書1本）
  • 有料サービスの最小MVP（最も需要が確認できた1機能から）
  • データソリューションの最小実証（1〜2社と小規模実証実験）
  • BCCヘルスケア事業との接続点の整理
アウトプット
  • 磨き上げ後の広場（Web・LINE）
  • 有料サービス試験運用レポート
  • データ実証レポート
  • 14期中間総括
判断ゲート（6月末） 有料化・データ販売それぞれの成立可能性を判定。15期で本格投資するか、見送るかの判断材料を揃える。
PHASE 3 魅せる 2027年7月 — 9月
（3ヶ月、14期 Q4）

14期の成果を「誰が見ても素敵な事業」として可視化し、15期以降への接続を作る。

主要アクション
  • 14期事業報告書の作成（社内向け・経営層向け）
  • 業界向けプレスリリース、事例発表、展示会出展
  • 15期事業計画と予算案の策定
  • 「次の3年」の中期ロードマップ案
  • 外部発信物の制作（業界誌寄稿、登壇、ホワイトペーパー）
アウトプット
  • 14期事業報告書
  • 15期事業計画案
  • 対外発信物一式
判断ゲート（9月末） 15期の投資判断と組織体制を経営決定。アプリ事業化／データ事業化のGo/No-Goを本判断する。
§ 04　判断基準

残す・削る・磨くは、
3つの基準で決める。

仕分けるフェーズで迷わないために、判断基準を先に合意しておく。コンテンツ・チャネル・キャンペーンなど、対象すべてに同じ3基準で○△×をつけることで、意思決定を情緒ではなく論理で進める。

01 利用ベース 直近12ヶ月のアクティブ率と運用工数（コスト）が見合っているか。コストの割に動かない資産は、まず候補に挙げる。
02 ブランドベース BCCのヘルスケア事業全体の中で、広場が担う固有のポジションを強化するか。他事業と重複する機能か、独自価値か。
03 未来ベース 14〜15期の事業の柱になりうるか、そこから派生する芽があるか。今は小さくとも、将来育つ可能性のあるものか。
